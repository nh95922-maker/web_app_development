# PRD — 任務管理系統

## 1. 專案概述
- **背景與動機**：現代人生活節奏快，常常面臨時間管理與任務追蹤的挑戰。本系統旨在提供一個簡單易用的任務管理工具，結合時間管控的概念，協助使用者有效安排時間、提升生產力。
- **目標用戶**：學生、上班族、自由工作者等需要管理個人時間與日常任務的使用者。
- **核心價值主張**：透過直覺的介面與時間排序功能，讓使用者能輕鬆掌控每日任務進度，減少遺漏與時間浪費，並清楚掌握完成狀態。

## 2. 功能需求
以下為系統主要功能及其對應的使用者故事：

1. **新增任務**
   - 作為使用者，我希望可以新增任務（包含任務名稱、預計完成時間等），以便記錄我的待辦事項。
2. **編輯任務**
   - 作為使用者，我希望可以編輯現有的任務內容與時間，以便在需求改變時即時更新任務資訊。
3. **刪除任務**
   - 作為使用者，我希望可以刪除不再需要的任務，以便保持任務清單的簡潔與精確。
4. **按時間順序排序**
   - 作為使用者，我希望系統可以自動按任務的期限或時間順序排序，以便我知道接下來該優先處理哪件工作。
5. **標記完成狀態**
   - 作為使用者，我希望可以將任務標記為「已完成」，以便追蹤個人進度並清楚區分未完成的項目。

## 3. 非功能需求
- **技術限制**：系統以 Flask 作為後端框架，搭配 Jinja2 樣板引擎進行畫面渲染，並使用 SQLite 作為輕量級關聯式資料庫。
- **效能與安全考量**：
  - 確保基本的防護措施（如防止 SQL Injection、XSS 等）。
  - 介面需確保 RWD 響應式設計，讓使用者在電腦或手機上均能順暢操作。

## 4. MVP 範圍 (Minimum Viable Product)
- **Must Have（必備）**：
  - 新增、編輯、刪除任務功能。
  - 基本的任務列表，並可標記完成/未完成狀態。
- **Should Have（應備）**：
  - 根據任務設定的時間，自動依時間順序排序清單功能。
- **Nice to Have（加分）**：
  - 任務即將到期時的視覺提示（如紅色字體或警告圖示）。
  - 各種過濾功能（例如：只看已完成、只看未完成）。
